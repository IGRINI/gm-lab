# GM-Lab — тестовый каркас: ГМ + NPC-субагенты (D&D 5e)

Стенд для архитектуры «ГМ-оркестратор + NPC как субагенты».
Дефолтный запуск использует **ChatGPT Codex OAuth**. Локальный **llama.cpp server**
и OpenAI-compatible провайдеры остаются опциями через `.env`/переменные окружения.

## Что показывает
Пишешь действие игрока — и видишь **всю механику** в правой панели:
- 🔧 **вызовы инструментов** ГМ: `ask_npc` / `roll_dice` / `get_world_fact` (нативные tool-calls);
- 🎭 **какой субагент** отвечает и его **скрытые мысли** (у каждого NPC свой контекст,
  карточка, секрет и память — ГМ ими не владеет);
- ♻ **ревью действия — делает САМ ГМ в своём треде**: получив черновик NPC, он либо
  описывает сцену (принял), либо вызывает `ask_npc` снова с полем `correction` —
  «нет, так нельзя, потому что…», и NPC переигрывает.

Слева — чистая «История» для игрока.

## Архитектура (коротко)
```
игрок → ГМ(tools) ──ask_npc──▶ NPC-субагент(JSON: reasoning/speech/action/claims)
            ▲   │  черновик возвращается ГМ как результат тула
            │   ▼
   ГМ В СВОЁМ ТРЕДЕ смотрит черновик (полный контекст сцены):
     • действие невозможно? → ask_npc(..., correction="почему нельзя") → NPC переигрывает
     • ок? → финальный нарратив игроку
```
Это generator–critic-петля, где **критик = сам ГМ**, а не отдельный stateless-judge:
ревью идёт **в треде ГМ** через ответ на тул-юз (correction-перевызов `ask_npc`), поэтому
у ревьюера полный контекст сцены — позиции, что произошло, преемственность.

Принципы:
- **правила/кубы — в коде** (`world.roll`, детерминированный сид);
- **секреты — только в карточке NPC**, ГМ их не держит (анти-метагейминг); защиту секрета
  ведёт сам NPC в промпте;
- **история** в `gm_messages` — OpenAI-формат (assistant с `tool_calls` + tool-результаты
  по `tool_call_id`); прошлые мысли (reasoning) в историю НЕ кладём;
- **у каждого NPC свой LLM-тред**: личная `npc_messages`-история, отдельный client/thread id
  для prompt-cache и свой компакт старых реплик; эта история не передаётся другим NPC;
- **JSON у NPC — свободным текстом**; все значения JSON (`reasoning`, `speech`, `action`,
  `claims`) просим писать по-русски, чтобы отладка была читаемой;
- **сэмплинг не трогаем** — параметры Gemma 4 уже в Modelfile.

## Файлы
| файл | что |
|------|-----|
| `config.py` | модель, бэкенд, лимит хопов, флаги думалки |
| `prompts.py` | системные промпты ГМ (он же ревьюер) и NPC |
| `world.py` | мир, карточки NPC, секреты, кубы, ограничения сцены |
| `rag.py` | локальный RAG: embedding client, SQLite cache, dense+keyword retrieval |
| `agents.py` | роли модели + инструменты (ask_npc с correction) |
| `gm_tools_guide.md` | как добавлять, искать, менять и удалять инструменты ГМ |
| `orchestrator.py` | цикл хода, NPC-треды и поток событий (генератор) |
| `server.py` + `index.html` | веб-интерфейс (SSE-стрим, карточки, раскрывашки) |
| `smoke.py` | CLI-прогон без интерфейса (принимает действие аргументом) |
| `llm_client.py` | клиент llama.cpp (OpenAI `/v1`, httpx) + мок-бэкенд |

## Запуск
```powershell
pip install -r requirements.txt

python server.py                                  # веб-интерфейс, Codex OAuth по умолчанию
$env:GM_BACKEND="llamacpp"; python server.py      # локальный llama.cpp
$env:GM_BACKEND="mock"; python smoke.py           # логика без модели
```
`smoke.py` принимает действие игрока аргументом — это CLI-вход для быстрых локальных
проверок без веб-интерфейса.

`config.py` читает `.env` рядом с проектом и не перетирает переменные, уже заданные в
PowerShell. Пример:

```dotenv
GM_BACKEND=codex
GM_CODEX_MODEL=gpt-5.4-mini
GM_HOST=0.0.0.0
GM_PORT=8000
```

## Веб-команды (строка ввода)
| команда | действие |
|---------|----------|
| `/constraint <текст>` | добавить ограничение сцены (по ним ГМ судит действия NPC) |
| `/event <текст>` | подбросить миру событие, которого NPC ещё не знают |
| `/reset` · `/quit` | новая партия · выход |

## Как увидеть, что ГМ возвращает действие на переделку
Спровоцируй невозможное действие. В сцене есть ограничения (`world.constraints`,
напр. «выход только через главную дверь, на виду у зала»). Надави на NPC так, чтобы он
захотел улизнуть/сделать скрытное — ГМ, увидев черновик, сам перевызовет `ask_npc`
с `correction` («задней двери нет…»), и NPC переиграет. Зависит от того, заметит ли
ГМ нестыковку (это его живое суждение, а не жёсткий чек) — можно подкинуть ограничение
прямо в игре через `/constraint`.

## Настройка
- Codex: `GM_BACKEND=codex`, модель `GM_CODEX_MODEL` (дефолт `gpt-5.4-mini`).
- llama.cpp: `GM_BACKEND=llamacpp`, сервер `LLAMA_HOST` (деф `http://localhost:8080`),
  модель `GM_MODEL`. Контекст (n_ctx) и сэмплинг задаются на старте llama.cpp-сервера.
  Лимит хопов за ход: `$env:GM_MAX_HOPS` (дефолт 6).
- Думалка: `NPC_THINK` (NPC, дефолт 0), `GM_THINK` (ГМ, дефолт 1).
  Видимые в лабе мысли/заметки просим писать по-русски. Управляется `enable_thinking` у Qwen.
- Компакт истории: `GM_HISTORY_TOKENS` (100000), `GM_KEEP_TURNS` (3), `GM_EVENTS_CAP` (400).
- Компакт NPC-тредов: `NPC_HISTORY_TOKENS` (64000), `NPC_KEEP_EXCHANGES` (6).
  Для 32k-контекста лучше запускать с `NPC_HISTORY_TOKENS=24000`, чтобы оставить место
  под карточку NPC, текущий scene slice и свежие наблюдения.
- RAG/факты мира: `GM_RAG_ENABLED=1` (дефолт), `GM_RAG_EMBEDDINGS_URL`
  (`http://127.0.0.1:8080/v1/embeddings`), `GM_RAG_EMBEDDINGS_MODEL`
  (`qwen3-embedding-4b-q4`), `GM_RAG_CACHE_PATH` (`gm_lab_embeddings.sqlite3`).
  В индекс попадают только actor-safe документы: публичные факты, текущая сцена,
  местонахождение NPC и услышанные игроком свидетельства/слухи. Hidden canon и секреты
  NPC не индексируются.
- Добавить NPC: карточка в `world._npcs()` — инструмент ГМ соберётся из мира автоматически.
